\section{Supplementary Methods}

\subsection{Software scope and release organization}

%  补充材料开头必须先定义软件范围。正确重点是 package-native workflow、optional model-backed routes、retained resources 三层边界，避免 reviewer 误解为“旧模型包装”或“新模型论文”。
TomatoPromoterDesigner is an installable Python command-line framework for tomato promoter sequence processing, motif annotation, tissue-associated scoring, motif-aware candidate design, report generation and figure export. The release is organized around a directly runnable package-native workflow, with optional model-backed routes and retained result resources provided for reproducibility. Package-native modules include deterministic motif scanning, an operational four-tissue scorer, motif-preserving candidate generation, tabular output writing and compact report generation. Optional model-backed routes and repository-retained result tables were generated during the same tomato promoter engineering project and are used for manuscript-level examples, supplementary figures and quantitative traceability checks.

%  这一段是防御性边界说明。要强调 DNABERT/MpraVAE/DeepSEED 是 route-specific software components，不是本文比较模型性能的对象。
Package-native commands provide the directly inspectable workflow, while MpraVAE-, DNABERT- and DeepSEED-derived resources expose route-specific model or post-processing behavior. The Application Note evaluates their integration and reproducibility as software components rather than comparative model performance. Component boundaries are summarized in Table~\ref{tab:component-boundary}.

\subsection{Reproducible resources and result reconstruction}

%  这一段说明数据和 retained result pack 的组织方式。正确写法是 provenance/reconstruction，不要让 external resources 看起来像缺失数据。
Repository-distributed data were organized into raw, processed and result-facing resources. Small tabular files required for manuscript reproduction and lightweight checkpoints used by explicit model adapter commands were retained in the repository, whereas large genomes, HDF5 corpora, BLAST databases and heavyweight optional resources are documented through external provenance manifests. The manuscript-facing retained result pack is generated by:

\begin{verbatim}
make reproduce-results
\end{verbatim}

%  这里给出可复制命令。投稿前必须保证 Makefile 中真实存在该 target；这个命令支撑 supplementary figures 和 retained tables 的复现。
This command executes the repository result-reproduction script and writes synchronized tables and figures under \texttt{data/results/reproducible\_legacy/}. The generated manifest records the source table and reconstructed output for each retained result. Major data resources used by the supplementary analyses are summarized in Table~\ref{tab:data-resources}. Image file locations are intentionally not listed as a formal table because the figures themselves are part of the submitted supplement; repository image paths are maintained in the project documentation rather than treated as scientific results.

\subsection{Command-line workflow}

%  这一段是软件说明核心。Application Note 的 supplement 要让 reviewer 看到用户如何从 FASTA 输入跑到 annotation、scoring、design 和 report。
The basic workflow begins with promoter sequences in FASTA format and passes them through annotation, scoring, design and report-generation steps:

\begin{verbatim}
pip install -e .

tomato-promoter-designer copy-example \
  --output outputs/demo_input.fasta

tomato-promoter-designer validate-input \
  --input outputs/demo_input.fasta

tomato-promoter-designer annotate \
  --input outputs/demo_input.fasta \
  --output outputs/demo_annotate.csv

tomato-promoter-designer predict \
  --input outputs/demo_input.fasta \
  --output outputs/demo_predict.csv

tomato-promoter-designer design \
  --input outputs/demo_input.fasta \
  --target fruit \
  --candidates 3 \
  --seed 42 \
  --output outputs/demo_design.csv

tomato-promoter-designer report \
  --input outputs/demo_design.csv \
  --output outputs/demo_report.json
\end{verbatim}

%  这一段说明同一 CLI 也有 optional adapter commands。必须把 demo FASTA 和 retained result pack 区分开：前者测试安装，后者支撑稿件图表。
The same interface exposes adapter-oriented commands for DNABERT-derived motif processing, MpraVAE four-tissue scoring and design, DeepSEED scalar scoring, figure export and model-figure reconstruction (Table~\ref{tab:commands}). The small bundled FASTA example is intended for software testing and documentation. Broader supplementary result summaries use the retained result pack described above.

\subsection{Input validation and sequence normalization}

%  这一段回答工具输入是什么。要明确普通 package-native route 可处理一般 promoter FASTA，而 MpraVAE checkpoint route 需要 165-bp unambiguous A/C/G/T，这是模型输入形状限制。
FASTA records are parsed into sequence identifiers and sequence strings. Empty FASTA files are rejected, empty sequence records are rejected, sequence strings are uppercased, duplicate identifiers are rejected and unsupported symbols outside \texttt{A/C/G/T/N/M} are not accepted. The ambiguous symbols \texttt{N} and \texttt{M} are accepted by the package-native validation layer only to allow inspection of retained or user-provided promoter records containing limited ambiguity codes; they are not introduced by the design workflow. The MpraVAE checkpoint-backed routes require canonical 165-bp unambiguous \texttt{A/C/G/T} sequences because this is the input shape expected by the bundled model architecture.

\subsection{Package-native motif annotation}

%  这一段解释默认 motif scanner。必须说明 exact matches 是 operational motif features，不是实验验证 TF binding sites，否则 reviewer 会追问 motif 生物学证据。
The package-native motif module performs deterministic exact-match scanning using the repository-default motif set \texttt{CAAAA}, \texttt{CTATT}, \texttt{ATTTT}, \texttt{TTAAA} and \texttt{TTTAT}. These short sequences were selected as a compact operational motif-feature set from recurrent short patterns retained by the project DNABERT-derived motif workflow and from motifs needed by the package-native motif-protection examples. They are not presented as an exhaustive tomato cis-regulatory vocabulary, a frequency-ranked motif panel or experimentally confirmed transcription-factor binding sites. For every occurrence, including overlapping occurrences, the scanner reports the sequence identifier, motif sequence, zero-based start coordinate, zero-based end coordinate and a match score equal to motif length. This module provides a transparent baseline regulatory-interpretation layer and a consistent input to the motif-aware design route.

\subsection{Optional DNABERT-derived motif post-processing}

%  这一段明确 DNABERT route 的边界：它处理 precomputed attention/table resources，不从 FASTA 重新跑 DNABERT inference；这是 optional post-processing，不是新 DNABERT 方法。
The DNABERT-derived adapter is exposed as \texttt{annotate-dnabert}. It processes precomputed DNABERT sequence and attention-score resources and produces motif-summary, labeled-sequence and metadata outputs \cite{Ji2021}. It does not run DNABERT inference from FASTA input; rerunning that upstream inference step requires a separately supplied fine-tuned checkpoint. In the retained post-processing route, high-attention sequence segments are merged into motif groups using the repository defaults implemented by the adapter: minimum motif length 5, window size 24, adjusted \(P\)-value cutoff 0.005 and minimum retained motif count 3. These thresholds are fixed reconstruction parameters for the retained motif table rather than newly optimized motif-discovery criteria. The retained processed motif table is used to summarize the top 20 motif instances in Fig.~\ref{fig:supp-motif}. This route is described as model-derived motif post-processing because the Application Note focuses on software integration and reproducibility rather than introducing a new DNABERT architecture.

\subsection{Tissue-associated scoring modules}

%  这一段说明 package-native scorer。关键是 operational/deterministic/not calibrated，不能让人误解成真实表达预测模型。公式前必须说明固定权重只是软件默认规则，不是训练得到的生物学参数。
TomatoPromoterDesigner reports continuous tissue-associated scores for root, stem, leaf and fruit rather than binary tissue labels. The package-native scorer is a deterministic operational module that combines GC ratio, AT ratio, poly-A content and default motif counts into four tissue-associated scores so the installed package remains directly runnable without external checkpoints. The terms in this scorer were chosen to use only transparent sequence features available from the installed package: global base composition, poly-A content and counts of the repository-default motif set. The numerical coefficients are fixed release defaults that define a reproducible fallback scoring route and make motif-count terms visible in route-internal ranking; they were not fitted from expression data and should not be interpreted as biological effect sizes, tissue-specific regulatory weights or evidence for promoter activity. Let \(G_C\) be GC ratio, \(A_T\) be AT ratio, \(P_A\) be normalized \texttt{AAAAA} count and \(c_m\) be the count of motif \(m\). The package-native scoring rules are defined as:

\[
\begin{aligned}
S_{root} &= 4.2G_C + 2.0c_{\mathrm{CTATT}} + 0.5c_{\mathrm{TTAAA}},\\
S_{stem} &= 3.2A_T + 1.8c_{\mathrm{TTTAT}} + 0.6c_{\mathrm{ATTTT}},\\
S_{leaf} &= 2.5G_C + 1.1c_{\mathrm{CAAAA}} + 0.7c_{\mathrm{CTATT}},\\
S_{fruit} &= 3.5A_T + 2.2c_{\mathrm{CAAAA}} + 1.3c_{\mathrm{ATTTT}} + 2.0P_A.
\end{aligned}
\]

%  公式后必须再次说明这些 score 只是 route-internal ranking values，不是 calibrated expression estimates；MpraVAE 和 DeepSEED 的输出也不能跨 backend 直接比较。
The preferred tissue is assigned as the tissue with the highest reported score. These operational heuristic scores are intended for workflow execution and route-internal candidate ranking, and are not calibrated expression estimates. For sequences that match the expected 165-bp input format, the explicit MpraVAE route can emit four-tissue model scores using the bundled checkpoint. These two routes use the same four tissue field names (Table~\ref{tab:output-fields}), but their numerical values are route-specific and should not be compared across backends as if they shared a common calibrated scale. The DeepSEED adapter is a separate scalar route and reports \texttt{predicted\_log2\_expression} and \texttt{predicted\_linear\_expression} together with a backend identifier; it does not emit four tissue scores.

\subsection{Optional MpraVAE-compatible training entry point}

%  这一段说明仓库不是只有 .pth 文件，也包含可检查的训练入口和模型结构。写法要强调 checkpoint compatibility，不要把它包装成本文新训练 benchmark。
The repository includes an inspectable training entry point for producing checkpoints compatible with the MpraVAE scoring and design adapters. The training loader reads unambiguous 165-bp sequences from \texttt{realB} and four continuous targets from \texttt{expr\_tissue\_1}--\texttt{expr\_tissue\_4}; records with invalid lengths, ambiguous bases or non-numeric targets are excluded. The default configuration uses a 64-dimensional latent representation, a seeded 90:10 random training/validation split, batch size 64, Adam optimization with learning rate 0.001 and 20 epochs. The objective combines sequence reconstruction loss, four-target mean-squared prediction loss and VAE Kullback--Leibler divergence with weights 1.0, 1.0 and 0.001, respectively. These training settings are documented compatibility defaults for producing adapter-compatible checkpoints and smoke-testable training runs; they are not presented as a hyperparameter optimization study. The checkpoint with the lowest validation loss is written as a \texttt{JointPromoterModel} state dictionary. This entry point documents and tests checkpoint compatibility; the bundled \texttt{best\_val\_corr\_model.pth} remains the retained checkpoint used for the manuscript-facing model route.

\subsection{Motif-aware candidate design}

%  这一段说明默认设计路线。正确重点是 motif protection、seeded substitutions、candidate ranking；不要写成 AI de novo design，也不要承诺设计结果真实有效。
The package-native design route first annotates motif positions in each input promoter and treats those positions as protected. It then generates a candidate pool by applying seeded substitutions to non-protected positions according to fixed target-tissue nucleotide-preference tuples implemented in the source code. These tuples are operational defaults that bias sampling toward GC-rich choices for root/leaf-oriented routes and AT-rich choices for stem/fruit-oriented routes, matching the transparent sequence features used by the package-native scorer; repeated bases in a tuple act as simple sampling weights. The default mutation rate is 0.18 per non-protected position and is used to generate visible candidate diversity while retaining detected motif positions. The mutation rate and nucleotide-preference tuples are deterministic release defaults for candidate generation, not optimized biological design parameters. Candidate pools are scored by the scoring layer and ranked by the requested target-tissue score. For each candidate, the output table records the original sequence, designed sequence, target tissue, candidate rank, four tissue-associated scores, preserved motif summary, design status and number of point differences.

%  这一段说明 MpraVAE design adapter 是 checkpoint-dependent optional route，不是 package-native 默认设计；必须与上一段区分清楚。
The MpraVAE design adapter provides a checkpoint-dependent route through the bundled lightweight checkpoint. It encodes a 165-bp promoter, optimizes the latent representation toward the target-tissue objective, decodes candidate sequences, applies promoter-like quality control and reports ranked unique candidates. This route is included as a model-backed design option and is not required for the package-native baseline workflow.

\subsection{Report generation and figure export}

%  这一段说明 report/figures 命令与图表复现。重点是从 standardized result tables 重画，而不是孤立图片文件。
The \texttt{report} command builds a compact JSON summary from a design table, including row counts, unique designed sequences, target tissues, mutation statistics and best-candidate summaries. The \texttt{figures} and \texttt{model-figures} commands export figure summaries from standardized result tables. The supplementary figures were redrawn from retained source tables so that their displayed values correspond to repository-distributed tabular resources rather than isolated image assets.

\subsection{Retained quantitative reference calculation}

%  这一段是 n=12,575 和 r=0.852 的防御性说明。必须写清楚它是 retained table 的 traceability statistic，不是独立测试集，也不是当前模型性能。
The retained quantitative reference table is \texttt{data/raw/mpravae/generated\_prediction\_results.csv}, exported during the project MpraVAE modeling workflow. It contains paired scalar-expression columns \texttt{Predicted\_Expr} and \texttt{True\_Expr}. Each row represents one predicted/true scalar-expression pair for a generated promoter sequence and its paired reference sequence; therefore \(n=12{,}575\) denotes valid expression-value pairs rather than necessarily 12,575 independent biological promoters. The table records the final exported values but does not contain stable promoter identifiers or train/validation membership, and the script that originally exported this table is not part of the current release. Consequently, the statistic is used only to verify the retained numerical table and reconstruct its figure, not to establish dataset independence or model generalization. Rows with missing, non-numeric or non-finite values in either value column were excluded. No log transformation or additional normalization was applied before the reported statistic. Pearson correlation was calculated on 12,575 valid pairs:

\[
r =
\frac{\sum_i (x_i-\bar{x})(y_i-\bar{y})}
{\sqrt{\sum_i (x_i-\bar{x})^2}\sqrt{\sum_i (y_i-\bar{y})^2}},
\]

%  统计结论只能写 retained quantitative traceability resource，不能写 model achieved r=0.852。
where \(x_i\) and \(y_i\) denote retained predicted and true scalar-expression values, respectively. The resulting value was \(r = 0.852097\), reported as Pearson \(r = 0.852\). This retained table is used as a quantitative traceability resource and figure-reconstruction anchor, not as an independent benchmark of scoring components in the current release.

\section{Supplementary Retained Analyses}

\subsection{Retained quantitative reference}

%  结果小节 S1 对应 Fig S1。这里只总结 retained reference 的范围和相关性，不扩大为模型性能评价。
The retained quantitative reference table contains 12,575 valid predicted/true scalar-expression pairs from \texttt{data/raw/mpravae/generated\_prediction\_results.csv}. Predicted values range from approximately \(-7.04\) to \(7.07\), whereas true values range from approximately \(-5.99\) to \(8.53\). Predicted and true values showed a strong positive correlation (Pearson \(r = 0.852097\); Fig.~\ref{fig:supp-quantitative}). This supports the use of the retained table as a quantitative reference for software-level traceability and figure reconstruction.

\subsection{Visualization subset for tissue-associated training targets}

%  结果小节 S2 对应 heatmap。必须写 visualization subset，不写 classification dataset 或 validation set；preferred_tissue 是 score-derived grouping。
The retained heatmap source table contains 100 tomato promoter sequences and four tissue-associated training-target values per sequence. These values originate from the \texttt{expr\_tissue\_1}--\texttt{expr\_tissue\_4} columns of the project training table and are not outputs of the package-native heuristic scorer. The subset was selected for visualization with 25 promoters each assigned to fruit, leaf, root and stem according to the largest retained training target. Target-margin values range from 1.624 to 6.034, with a median of 3.6785 and a mean of 3.3995. The heatmap shows visible block structure after grouping by dominant target (Fig.~\ref{fig:supp-expression}), illustrating the multi-tissue structure of the retained training table without treating the subset as a benchmark dataset.

\subsection{Retained motif summary from optional motif-analysis workflow}

%  结果小节 S3 对应 motif figure。这里展示 retained motif patterns 作为检查资源，不能写成实验确认的 TFBS 发现。
The retained optional motif-analysis summary reports the 20 most frequent motifs after DNABERT-derived post-processing. The five most frequent retained motifs are \texttt{TACTT} (1,893 instances), \texttt{CTATT} (1,264 instances), \texttt{ATTTT} (841 instances), \texttt{CAAAA} (722 instances) and \texttt{TTTAT} (384 instances). These results provide a compact motif-level view that complements the package-native exact-match motif scanner (Fig.~\ref{fig:supp-motif}).

\subsection{4-mer similarity between original and generated promoters}

%  结果小节 S4 对应 k-mer comparison。只能解释为 sequence-composition similarity/QC，不能写成设计成功或功能保持。
The retained 4-mer comparison table contains 256 4-mers for each of three regions: the full promoter sequence, the proximal 20-bp segment and the distal 20-bp segment. The 20-bp regional windows are retained summary windows used for compact sequence-composition inspection; they are not presented as experimentally defined promoter boundary annotations. Correlation between original and generated 4-mer frequencies is strongest in the proximal 20-bp region (\(r = 0.835\)), followed by the full promoter region (\(r = 0.767\)) and the distal 20-bp region (\(r = 0.585\)). These results indicate that retained generated sequences preserve substantial sequence-composition structure, with regional differences visible in the reconstructed panels (Fig.~\ref{fig:supp-kmer}).

\subsection{Retained fruit-targeted design candidates}

%  结果小节 S5 对应 design candidate figure。必须强调 MpraVAE-derived latent route，且 Hamming difference 很大时不能说 promoter structure preserved，只能说 retained adapter-derived design-output view。
The retained design summary table contains 199 fruit-targeted promoter candidates from the MpraVAE-derived latent design route. Predicted fruit-associated scores range from 1.7999 to 18.4738, with a median of 6.4398 and a mean of 6.7601. Fruit-associated bias margins range from 0.8936 to 2.9709, with a median of 1.8274 and a mean of 1.8522. Hamming differences between original and latent-decoded generated sequences range from 82 to 131, with median and mean values both equal to 105. These distributions provide a retained adapter-derived design-output view that is distinct from the package-native seeded substitution route described in the Methods (Fig.~\ref{fig:supp-design}).
