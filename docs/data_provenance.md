# Data Provenance and Redistribution Notes

The repository now ships with a curated `data/` layer so that manuscript-facing examples, processed tables, and reconstructed figures can be inspected without reopening the full legacy project folders. That bundled layer is intentionally selective. We included small tables and figures that are directly useful for software demonstration, reproducibility checks, and manuscript preparation, while leaving large genomes, HDF5 corpora, BLAST databases, and heavyweight checkpoints outside normal git history.

The bundled files are not all equivalent from a licensing perspective. `DNABERT` is the clearest case in the local workspace because an Apache-2.0 license file is present in the upstream snapshot. That gives us a concrete basis for retaining attribution when we redistribute the bundled `DNABERT`-derived support files and processed motif outputs. By contrast, the local `MpraVAE` snapshot includes a project `README` but no standalone license text, so its bundled raw tables and copied figure assets should be treated as provenance-preserved research materials rather than automatically publication-ready open data. The situation is even less clear for `deepseed`, because the root `LICENSE` file in the local snapshot appears malformed and does not currently provide an interpretable software license statement.

For that reason, the safest public-release strategy is tiered. The repository-native demo outputs in `data/results/demo/`, the modern SVG figures in `data/results/figures_*`, and the code-generated documentation files are the easiest artifacts to expose broadly because they are generated inside the unified package. The mixed-provenance legacy figure bundle is still valuable for the manuscript story, but it should travel with explicit provenance tables and, before a public GitHub or Zenodo release, with a final review of whether each copied upstream asset is permitted to be redistributed in that form.

The operational rule for this repository is therefore simple. Use `data/source_registry.tsv` as the first-pass source and licensing table, use `data/inventory.tsv` for file-level provenance, and use `data/external/large_files.tsv` for the large artifacts that remain outside git. Before the final public release that accompanies the `Bioinformatics` submission, replace any ambiguous copied files with regenerated derivatives when possible, or document written permission and attribution if the original asset must remain bundled.
